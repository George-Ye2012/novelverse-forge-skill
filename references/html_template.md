## Architecture Change

### v3 (OLD): Preset Theme System
```
template.html → {{BASE_CSS}} + {{THEME_CSS}} + {{DARK_CSS}} + {{INLINE_JS}}
                     ↑              ↑             ↑
               base.css      themes/xxx.css   dark.css  (6 fixed themes)
```

### v4 (NEW): Dynamic Design Genome System
```
template.html → {{BASE_CSS}} + {{DYNAMIC_CSS}} + {{INLINE_JS}}
                     ↑              ↑
               base.css      generated by generate_html.py
               (layout only) from visual_genome.py design.json
```

**Key differences:**
- **No `assets/css/themes/` directory** — all themes deleted
- **No `assets/css/dark.css`** — dark mode handled by base.css color-mix overrides
- **No `{{THEME_CSS}}` or `{{DARK_CSS}}` placeholders** — replaced by `{{DYNAMIC_CSS}}`
- **`{{GOOGLE_FONTS_URL}}`** is now dynamically generated (only loads fonts actually used)
- **`{{DESIGN_RATIONALE_FOOTER}}`** is a new placeholder for the design explanation

## Page Structure

```html
<!DOCTYPE html>
<html lang="zh-CN" data-theme="light">
<head>
  <!-- Google Fonts (dynamic, only loads used fonts) -->
  <link href="{{GOOGLE_FONTS_URL}}" rel="stylesheet">

  <!-- Base CSS: layout skeleton, components, reset -->
  <style>{{BASE_CSS}}</style>

  <!-- Dynamic CSS: ALL visual styles generated from design scheme -->
  <style>{{DYNAMIC_CSS}}</style>

  <!-- Minimal inline styles: page transitions, overlay, nav indicators -->
  <style>/* SPA transitions & quick-nav styles */</style>
</head>
<body class="theme-dynamic">

  <nav id="main-nav"><!-- Navigation with conditional links --></nav>

  <main id="app">
    <section id="page-overview"><!-- Hero section --></section>
    {{PAGE_TIMELINE}}     <!-- Timeline page (if events exist) -->
    <section id="page-characters"><!-- Entity gallery --></section>
    {{PAGE_FACTIONS}}     <!-- Factions page (if factions exist) -->
    {{PAGE_BEINGS}}       <!-- Beings page (if creatures exist) -->
    {{PAGE_RELATIONS}}    <!-- Relations page (if relationships exist) -->
    {{PAGE_INTERACTIVE}}  <!-- Interactive/mysteries page (if low-conf entities exist) -->
  </main>

  {{DESIGN_RATIONALE_FOOTER}}  <!-- NEW: Design explanation footer -->

  <!-- Data + JS + Bootstrap -->
</body>
</html>
```

## Dynamic CSS Generation

The `{{DYNAMIC_CSS}}` block is generated entirely by `generate_html.py` from the
visual design scheme JSON. It contains:

### Section 1: CSS Custom Properties (Design Tokens)
```css
:root {
  /* Color System — 20+ variables from design.color */
  --nv-color-primary: #...;
  --nv-color-secondary: #...;
  --nv-color-accent: #...;
  --nv-color-bg-start: #...;
  --nv-color-bg-end: #...;
  --nv-color-text-primary: #...;
  --nv-color-text-secondary: #...;
  --nv-color-surface: #...;
  --nv-color-border: #...;
  --nv-color-glow: #...;
  /* ... etc */

  /* Typography — from design.typography */
  --nv-font-heading: 'FontName', fallback;
  --nv-font-body: 'FontName', fallback;
  --nv-heading-weight: 700;
  --nv-heading-letter-spacing: 0.02em;
  --nv-line-height-base: 1.8;
  /* ... etc */

  /* Spacing — from design.space */
  --nv-space-2: 8px;
  --nv-space-4: 16px;
  --nv-space-6: 32px;
  /* ... etc */

  /* Radius — from design.space */
  --nv-radius-xs: 4px;
  --nv-radius-md: 12px;
  --nv-radius-lg: 16px;
  /* ... etc */

  /* Shadows — from design.lighting */
  --nv-shadow-sm: 0 2px 8px rgba(...);
  --nv-shadow-lg: 0 16px 48px rgba(...);
  /* ... etc */

  /* Transitions — from design.motion.speed */
  --nv-duration-fast: 0.15s;
  --nv-duration-base: 0.25s;
  /* ... etc */
}
```

### Section 2: Texture & Background
Generated CSS for the body background texture overlay (SVG filters, gradients,
etc.) based on `design.texture.type` and `design.texture.intensity`.

### Section 3: Motion & Animation Keyframes
Generated CSS keyframes for glow-pulse, parallax-drift, wave-motion, etc.
based on `design.motion.type` and `design.motion.speed`.

### Section 4: Theme-Aware Component Overrides
Overrides for component styles that need design-specific values:
- Hero title gradient (primary → accent)
- Section header accent bar color
- Entity card hover glow color
- Timeline track gradient
- Scrollbar thumb gradient
- Button gradient and glow
- Active nav/filter indicator colors

## Design Rationale Footer

The `{{DESIGN_RATIONALE_FOOTER}}` block renders a footer section that explains
every design decision to the user:

```html
<footer class="design-rationale-footer">
  <div class="rationale-inner">
    <h3>💡 设计理念 · Design Rationale</h3>
    <p>🎨 从文中反复出现的「血红」「暗金」等色彩意象中提取了主色调</p>
    <p>🔤 从文中的古代帝国气质，选择了具有古典感的衬线字体「Noto Serif SC」</p>
    <p>🖼️ 从文中场景分布提取了「水墨」纹理质感</p>
    <p class="design-concept">「以史诗感为基调，构建与文本气质共鸣的视觉体验」</p>
    <p style="font-size:0.7rem;opacity:0.5;">
      Design: 「深渊回响·暗金纪元」 · design_id: a1b2c3d4
    </p>
  </div>
</footer>
```

This makes the design process **transparent** — users can see that every
visual choice has a specific origin in their text.

## Google Fonts Dynamic Loading

In v4, the Google Fonts URL is dynamically generated to **only load fonts
actually used** in the design scheme:

```
https://fonts.googleapis.com/css2?
  family=Noto+Serif+SC:wght@400;700;900
  &family=Inter:wght@300;400;500;600;700;800;900
  &display=swap
```

Font weights are selected based on:
- `typography.heading_weight` → appropriate weight range for heading font
- `typography.body_weight` → appropriate weight range for body font
- `typography.decorative_font` → included if set

The font configs map in `generate_html.py` ensures each font loads with
the correct weight ranges.

## Module Specifications

### Hero Section
- Full-viewport (min-height: 100vh) with gradient overlay
- Title in heading font at display size with gradient text effect
- Stats counters animated on scroll
- Quick-nav circle buttons for available pages
- Scroll-down indicator with bounce animation

### Entity Gallery
- CSS Grid: `repeat(auto-fill, minmax(300px, 1fr))`
- Type filter tabs: All / Characters / Factions / Creatures
- Cards: glass-morphism with hover glow
- Low-confidence cards: dashed border + TENTATIVE badge
- Expandable source evidence blocks

### Timeline
- Horizontal scroll with drag-to-pan
- Era filter buttons
- Zoom in/out controls
- Alternating card positions (odd/even)
- Contradiction-flagged events: warning border

### Relationship Web
- Text-based card list (no D3.js dependency)
- Color-coded by relationship type
- Legend with type counts
- Hover: slide right + glow border

### Search
- Full-screen overlay triggered by Ctrl+K or nav button
- Real-time filtering across all entity types
- Keyboard navigation (↑↓Enter)
- Results show type icon, name, page hint

## Accessibility Requirements

- WCAG 2.1 AA: text/background contrast ≥ 4.5:1
- Keyboard navigation: all interactive elements focusable
- Screen readers: semantic HTML5 landmarks, alt text on images
- Skip link: first focusable element
- Reduced motion: `prefers-reduced-motion` disables all animations
- Dark mode: toggle with `aria-pressed`, localStorage persistence
